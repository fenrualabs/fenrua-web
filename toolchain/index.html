VERSION VERIFICATION METHOD

What the registry proves

The registry proves that a bounded command or repository inspection was performed and that the normalized result was published. It does not prove a tool was used in a security campaign unless the row marks evidence production and points to the relevant path.

Installed: a binary, container image, or project dependency was detected.

Executed: a read-only version, help, list, or repository inspection command was run.

Evidence-producing: the tool participates in an existing repeatable evidence path in this repository.

Exploratory: available or inspected, but not part of the canonical verification pipeline.

Unavailable: the expected command or dependency was not detected locally.

Known limitation: contract evidence refresh is pending; no stale deployment, bytecode, reserve, or invariant claim is finalized here.

Evidence-lock integrity: dependencies and tool versions are preserved to match the inspected evidence environment; no post-evidence updates are implied.
