# Fenrua Public Interface Doctrine

Status: claim-control doctrine  
Last reviewed: 2026-07-12

## Purpose

The public website is an evidence interface for Fenrua's Layer 0 AI security
infrastructure. It must help a technical visitor understand what exists, what is
planned, what has evidence, and what remains unverified.

## Public Truth Boundary

The website must never imply that:

- Installed tooling automatically proves security.
- A local test proves production safety.
- A source audit proves deployed bytecode.
- A compiled contract proves a live deployment.
- A schema proves an implemented API.
- A research result proves production certification.
- A chain observation proves chain safety.
- An internal score proves external assurance.
- A public evidence record proves all future revisions.

## Required Claim Shape

Every significant claim must include:

- Source.
- Evidence path.
- Freshness date.
- Maturity label.
- Scope.
- Limitation or non-claim.

## Maturity Labels

Allowed maturity values:

- Research.
- Specification.
- Reference implementation.
- Prototype.
- Testnet.
- Read-only live.
- Gated beta.
- Production.
- Paused.
- Retired.
- Unavailable.

## Interface Rules

- No token-first or wallet-first positioning.
- No fake application launch controls.
- No fabricated live verification.
- No stale contract detail.
- No private environment data.
- No raw local paths, usernames, machine names, endpoints, tokens, or secrets.
- No remote tracking scripts.

## Public Navigation

Primary navigation should map to user intent:

1. Overview.
2. Architecture.
3. Security Kernel.
4. Utilities.
5. Research.
6. Verify.
7. Developers.
8. Toolchain.
9. Evidence.
10. Status.
