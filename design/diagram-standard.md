# Fenrua Diagram Standard

Status: Active public-interface standard v0.1
Owner: Architecture and Public Platform
Last reviewed: 2026-07-14

Public diagrams are semantic, keyboard-readable ordered structures with a text
equivalent. Every node carries a state label such as `Current`, `Reference`,
`Specification`, `Planned`, `Optional external`, or `Agreement-specific`.

Diagrams describe interfaces and boundaries, not deployment topology, service
availability, customer tenancy, private keys, or runtime throughput unless a
separate record provides that exact evidence. Arrows and sequence ordering must
never imply that an optional control plane, chain observation, or public site
is a mandatory authorization dependency.

Use functional borders, restrained surfaces, and text labels. Do not use
decorative node constellations, animated link effects, or abstract 3D graphics.
